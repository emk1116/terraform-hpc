# Titan HPC Cluster

Production-style HPC cluster on AWS using Slurm with autoscaling compute nodes, deployed entirely via Terraform.

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                      AWS VPC                         │
│                                                      │
│  ┌─────────────────────┐   ┌──────────────────────┐ │
│  │      Head Node       │   │   Compute Nodes      │ │
│  │  (t3.micro)          │   │   (t3.micro × 0–2)   │ │
│  │                      │   │                      │ │
│  │  • slurmctld         │   │  • slurmd            │ │
│  │  • slurmdbd          │   │  • munge             │ │
│  │  • MariaDB           │   │                      │ │
│  │  • munge             │   │  Launched on demand  │ │
│  │                      │   │  Terminated on idle  │ │
│  └─────────┬────────────┘   └──────────────────────┘ │
│            │ EIP (public)                             │
└────────────┼────────────────────────────────────────-┘
             │ SSH
           Operator
```

**Autoscaling**: Slurm's `ResumeProgram` launches EC2 instances when jobs are queued. `SuspendProgram` terminates them after 120 seconds idle. No instances run when the cluster is idle.

## Modules

| Module | Purpose |
|---|---|
| `network` | VPC, subnet, internet gateway, route table, security group |
| `iam` | IAM roles and instance profiles for head node and compute nodes |
| `head-node` | Head node EC2 instance, EIP, Slurm controller setup |
| `compute-fleet` | Launch template and ASG for compute nodes |

## Prerequisites

- Terraform >= 1.14.7
- AWS CLI configured (`aws configure`)
- An SSH key pair

## Quick Start

**1. Generate SSH key**
```bash
ssh-keygen -t ed25519 -f ~/.ssh/titan-hpc -C "titan-hpc"
```

**2. Create your vars file**
```bash
cp non-prod.tfvars.example non-prod.tfvars
# Edit non-prod.tfvars — set ssh_allowed_cidr to your IP:
#   curl ifconfig.me
```

**3. Deploy**
```bash
terraform init
terraform apply -var-file=non-prod.tfvars
```

**4. Connect**
```bash
ssh -i ~/.ssh/titan-hpc ec2-user@<head_node_public_ip>
```

**5. Submit a job**
```bash
sbatch ~/job.sh
squeue          # watch the queue
sinfo           # watch nodes spin up
sacct -j <id>   # check results after completion
```

**6. Destroy**
```bash
terraform destroy -var-file=non-prod.tfvars
```
> Compute nodes launched by Slurm are automatically terminated before VPC teardown.

## Configuration

All settings are in your `non-prod.tfvars`:

| Variable | Default | Description |
|---|---|---|
| `namespace` | `titan` | Prefix for all AWS resource names |
| `env` | `non-prod` | Environment tag |
| `ssh_allowed_cidr` | — | Your IP in CIDR format (`x.x.x.x/32`) |
| `head_node_instance_type` | `t3.micro` | Head node size |
| `compute_instance_type` | `t3.micro` | Compute node size |
| `max_compute_nodes` | `2` | Maximum concurrent compute nodes |
| `aws_region` | `us-east-1` | AWS region |

## Slurm Node Spec

Compute nodes are defined in `slurm.conf` as:
```
CPUs=2  RealMemory=900  State=CLOUD
```
Adjust these in the init script template if you change instance type.

## Cost

With `t3.micro` instances (~$0.0104/hr each):
- Head node runs continuously while deployed
- Compute nodes only run while jobs are active
- Idle cluster cost ≈ **$0.25/day** (head node only)

## Security

- SSH restricted to `ssh_allowed_cidr` only
- IMDSv2 enforced on head node
- EBS volumes encrypted
- Slurm DB password randomly generated by Terraform (never hardcoded)
- Munge key generated at boot, stored in SSM Parameter Store (SecureString)
- No secrets in code or `.tfvars` files (gitignored)
